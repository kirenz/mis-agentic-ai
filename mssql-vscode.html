5  VS Code

Visual Studio Code acts as your code editor, terminal, and database explorer. This chapter covers the essential extensions and workspace settings for SQL Server work.

5.1 Install VS Code

If you haven’t already, install Visual Studio Code:

  • macOS: Download the from code.visualstudio.com and drag Visual Studio Code into Applications.
  • Windows: Download the installer and click through the setup, leaving the default options enabled (add to PATH, register the code command, etc.).

Launch VS Code once to allow it to finish installing components.

5.2 Essential extensions

Open the Extensions view and install:

  1. Python (ms-python.python) – language support and linting.
  2. SQL Server (mssql) (ms-mssql.mssql) – database connections and query runner.
  3. vscode-icons (vscode-icons-team.vscode-icons) – file icons for better visual identification.

5.3 Enable the code command

5.3.1 macOS Terminal

  • Launch VS Code, press Cmd+Shift+P, and run Shell Command: Install ‘code’ command in PATH.

  • Close any open Terminal windows, then open a new one and run

    code --version

    to confirm the CLI is available.

  • From the folder you want to open, run

    code .

    to launch VS Code in that location.

5.3.2 Windows (Git Bash)

  • Run

    code --version

    in Git Bash; if it works, you’re done.

  • If the command is missing, add VS Code’s bin directory to your Git Bash PATH by appending the snippet below to ~/.bashrc (create the file if it doesn’t exist):

    VS_CODE="/c/Users/$USERNAME/AppData/Local/Programs/Microsoft VS Code/bin"
if [ -d "$VS_CODE" ] && [[ ":$PATH:" != *":$VS_CODE:"* ]]; then
  PATH="$VS_CODE:$PATH"
fi

  • Reload Git Bash

    source ~/.bashrc

Or open a new window and run

code --version

Again. Once it prints a version number,

code .

will open VS Code from that folder.

5.4 Integrated terminal tips

  • Open the integrated terminal in VS Code. On Windows, set Git Bash as the default shell so every VS Code terminal matches the rest of this guide.

  • Set your default terminal profile via Terminal > Configure Default Profile… so that VS Code opens the same shell you used during installation.
